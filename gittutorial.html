Git & GitHub

Overview about Git & GitHub

IMPORTANT: Git & GitHub are not the same


Git is similar to "CMD Promt" on Windows which is a version control tool

Github is a git site hosting repository

How to download Git

Simply open a broswer and search for Git

Select either Git or Download - Git

Press Downloads and choose your operating system (MAC, Window, Linux) and click download

Basic comands about Git

There are 4 basic Git command that you will have to use everytime


1. Git pull
To get the latest repository

2. Git add - (git add index.html)
To add file(s) to the index

3. Git commit - (git commit -m "a msg")
To commit change(s) that you have made

4. Git push
To push file(s) you just change to the repository

Resolving conflict
Merge Conflict

Merge Conflict happen when you merge branches that have competing commits, and git needs your help to decide which changes to incorporate in the final merge

In this situation you can choose to keep the origin line of code or accpet the new changes that was made by the others

NOTE: Git display multiples ways to resolve the conflict (Accept current change, Accept incoming change, Acpet both change,...)

Simply pick a change and you conflict will be resolve


Picture source "YouTube": Code 2020


TO AVOID HAVING THIS CONFLICT


1. Do not work on the same file(s)

2. Do not work on the same part(s)

3. Call your partners

4. Work on the same time as your partners

Branching & Merge

There are 7 basic Git branch command that you need to know


1. Git branch
To see how many branch(es) there are

2. Git staus
To see what branch you are in or using

3. Git branch "Name of the new branch"
To create a new branch(es)

4. Git checkout "Name of the new branch"
To switch between branch(es)

5. Git remote
To check the origin URL

6. Git push origin "Name of the new branch"
To push branch(es) to the repository

7. Git merge
To combine changes made on two distinct branches
